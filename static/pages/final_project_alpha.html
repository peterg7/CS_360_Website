    CS 360 - Final Project Alpha Release

    Project Overview

    This project worked with two datasets from Kaggle containing Tweets regarding the conflict between Russia and the Ukraine. The objective is to assess the accuracy of unsupervised sentiment analysis techniques on these datasets and build various visualizations accordingly. Several ML models were examined and the highest performance was earned by a Linear SVC. All of the models were trained on a dataset that contained Tweets with known sentiment (all containing patriotic hashtags for the Ukraine). The current visualizations include the distribution of sentiment found in every word contained in the dataset as well as each individual tweet. The other charts display the sentiment associated with the most common usernames and hashtags.

    Current Blockers

    At this stage, the largest challenge has been producing the data which will be used for visualizations. Pipeline #1 has been completed and runs smoothly but the issue lies in the project's nature itself. Because this task uses unsupervised learning, it has been difficult to produce consistent results from the sentiment analysis. In order to minimize the inconsitency in the data, there is a section in pipeline #1 that requests user input for assigning the sentiment values (negative, neutral, or positive) to each cluster (see Data Processing - Pipeline). However, this solution has not provided a significant improvement on reliability. There are other ways to improve the pipelines stability and there will be a patch by the next release.

    Another, much smaller, roadblock has been file organization. There are many moving parts to this project including Jupyter Notebooks, Python scripts which are used for the pipelines and all other processes for the sentiment analysis as well as HTML, CSS, and Javascript files used for the D3.js visualizations. This combination is not an issue as described but the complexity is due to the repeated output from the pipelines and the many data files that are produced. This issue is directly linked to the instability of the current pipelines which causes them to be run multiple times in order to produce valid results. Every execution produces new data files which pile up and make file management difficult. This issue should be resolved by the next release once the pipeline has been tuned and the results are consistent.

    Lastly, the dataset for the Slava Ukraini Tweets has stopped receiving daily updates. Furthermore, the most recent dataset produces results that are very different from the initial analysis. This has posed an issue for the training of models because the expected results are not the same which adds to the pipeline's inconsistency. The stall in dataset updates is not ideal but shouldn't pose a large enough issue to forsake Part 2 or Part 3 of the Project's Objective because there are still 15 versions of the dataset hosted on Kaggle.com.
