    Project Objectives

    Part 1

    1. What is the estimated, current sentiment of (presumably Ukranian patriot) "tweeters" when posting about the Ukraine, President Zelensky, and as an extension, the war with Russia (as identified using the #SlavaUkraini hashtag)?

    2. When looking at various features of Twitter, what evidence supports or rejects the above estimated sentiment (ie. usernames, hashtags, retweets, keywords,...)

    Part 2

    1. What is the inferred sentiment and accuracy when applying a predictive model trained on the #SlavaUkraini data to a more general dataset of tweets which involve Russia and/or Ukraine?

    2. (How have tweets changed over time?)

    Part 3

    1. Adding the broader dataset to the training set of the predictive model, what is the predicted sentiment of tweets regarding Russia and/or Ukraine at time t from the present date?

      • (What will the sentiment look like in the future?)

    Data Processing

    Data processing will be the majority component of the project. The data will be imported directly from Kaggle as a live link in order to always pull the most recent dataset and avoid the process of storing and referencing it locally. There will be 3 ETL pipelines to address each "Part" as described in the Project Objectives. These are outlined below

    Part 1 Pipeline

    Extract

    1. Import data into Pandas from Kaggle

    2. Select relevant features (date, user_name, retweets, text, hashtags)

    3. Clean text data (tweet text & hashtags)

      1. Remove punctuation and non alpha-numeric characters

      2. Lowercase all characters

      3. Split words on spaces

      4. Lemmatize words based on verbs

    4. Convert dates to datetime for use in comparisons and other mathematical operations

    5. Filter tweets to only include set of keywords ("ukraine", "russia", "zelensky")

    Transform

    1. Collect bigrams from tweet text

    2. Build a vector model trained on the bigrams

    3. Use a clustering algorithm (KMeans) to build 3 clusters from the vector model

    4. Map the clusters to be defined as neutral, positive, and negative (with user input)

    5. Reshape data into Pandas

    6. Determine sentiment of every word from the clusters

    7. Store sentiment of every tweet extracted from clusters

    Model

    1. Build training and testing datasets from the sentiment dataframe built above

    2. Transform new dataset text into vectors

    3. Fit a LinearSVC (linear support vector classification) with the training data

    4. Test performance of the SVC using the test data

    5. Fit a MultinomialNB (multinomial naive bayes) with the training data for performance comparison with the LinearSVC

    6. Test performance of the NB using the test data

    7. Compare performance of the SVC and NB

    8. Export the "higher performing" algorithm to be used in Part 2

    Load

    1. Save datasets & embeddings along pipeline to local directory

    2. Final sentiment dataframe exported to local directory for analysis/visualizations

    Part 2 Pipeline

    Extract

    • - Same as Part 1 Pipeline - Extract with the generalized dataset described in Data

    Transform

    1. Import vector model created in Part 1 Pipeline - Transform

    2. Build vectors from recently extracted data using the imported vector model

    3. Import the predictive model which was stored in Part 1 Pipeline - (Model)

    4. Make inferences using the model on the vectors built from the new dataset

    5. Map model outputs to sentiment encodings and original tweets

    Load

    1. Save inferred sentiment dataset to a local directory for analysis/visualization

    Part 3 Pipeline

    Extract

    • Same as Part 1 Pipeline - Extract but with both datasets described in Data (which will be merged into one)

    Transform

    • Same as Part 1 Pipeline - Transform with combined dataset

    • Group dataset by time to be used in a time-series predictive model

    Model

    1. Build highest performing model on the combined dataset just as in Part 1 Pipeline - (Model). But this time using a model optimized for a time-series:

      1. Linear Regression

      2. Dense Sequential

      3. CNN

      4. RNN

      5. LSTM

    2. Use combined dataset (or even large one collected from another source), generate inferences using the selected model for some time t in the future

    3. Export predictions to be used for analysis/visualizations

    Load

    • Same as Part 1 Pipeline - Load with combined dataset

    Related Works

    Research performed by other groups/organizations in the space of sentiment analysis. Some papers dealt specifically with Twitter data while others dealt with the task of unsupervised sentiment analysis with other source data. All of these papers are directly related to this project's overarching task of using Twitter data to infer the attitude of the public regarding the political and social crisis that is occuring between Russia and the Ukaraine.

    1. Wang, H., Can, D., Kazemzadeh, A., Bar, F., & Narayanan, S. (2012, July). A system for real-time twitter sentiment analysis of 2012 us presidential election cycle. In *Proceedings of the ACL 2012 system demonstrations* (pp. 115-120).

      • Wang et. al explore the use of sentiment analaysis on real-time Twitter data produced in response to significant news or events, specifically during the 2012 US Presidential election. The goal of the study is to provide instant analysis of the public's opinion of the election based on specific, related events.

    2. Go, A., Huang, L., & Bhayani, R. (2009). Twitter sentiment analysis. *Entropy*, *17*, 252.

      • Go et. al experiment with basic supervised learning sentiment analysis techniques using Twitter data. The purpose was to determine the accuracy of an algorithm which can be trained and used to predict the sentiment of a given query word.

    3. Mittal, A., & Goel, A. (2012). Stock prediction using twitter sentiment analysis. *Standford University, CS229 (2011 http://cs229. stanford. edu/proj2011/GoelMittal-StockMarketPredictionUsingTwitterSentimentAnalysis. pdf)*, *15*, 2352.

      • The big-picture of the paper is analyzing the relationship between the world of social media and the financial world. After determining the sentiment of the public using Twitter, the trained algorithm was used to predict the stock market movements.

    4. Fernández-Gavilanes, M., Alvarez-López, T., Juncal-Martínez, J., Costa-Montenegro, E., & González-Castano, F. J. (2015, June). Gti: An unsupervised approach for sentiment analysis in twitter. In *Proceedings of the 9th International Workshop on Semantic Evaluation (SemEval 2015)* (pp. 533-538).

      • The emergence of electrical cars has brought about mixed feelings by the public regarding efficiency, safety, and reliability. Using news articles and customer reviews, Everts and Jiang explore the public's sentiment of EV cars and attempt to determine their sources.

    5. Everts, J., & Jiang, X. (2021). Making sense of electrical vehicle discussions using sentiment analysis on closely related news and user comments. *arXiv preprint arXiv:2112.12327*.

      • Yu et. al research a way to identify students who are unlikely to pass a certain course. They combined early assessment work (homeworks, quizzes, etc.) with the results of sentiment analysis performed on student's self-evaluated comments.

    6. Bermingham, A., & Smeaton, A. (2011, November). On using Twitter to monitor political sentiment and predict election results. In Proceedings of the Workshop on Sentiment Analysis where AI meets Psychology (SAAIP 2011) (pp. 2-10).

      • Guda et. al inspect the performance of various machine learning techniques in their ability to predict the sentiment of restaurant reviews. Each algorithm is given the meta and text data associated with the reviews and is tasked with determining the overall sentiment of customers.
